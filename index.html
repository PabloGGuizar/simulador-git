Prepara tu Entorno de Desarrollo Real

VS Code es el editor de código más popular. Es donde escribirás y organizarás todos tus proyectos.

  1. Ve a la página oficial: code.visualstudio.com.
  2. El sitio detectará tu sistema operativo (Windows, Mac, Linux) y te ofrecerá el instalador correcto.
  3. Descarga y ejecuta el instalador, aceptando las opciones por defecto.

Node.js es un entorno que permite ejecutar JavaScript fuera del navegador. Es esencial para trabajar con React. Al instalarlo, también se instala npm (Node Package Manager), que usarás para manejar las dependencias de tu proyecto (como React o `gh-pages`).

  1. Ve a la página oficial: nodejs.org.
  2. Descarga la versión LTS (Long Term Support). Es la más estable.
  3. Ejecuta el instalador y sigue los pasos, aceptando las opciones por defecto.
  4. Para verificar que se instaló correctamente, abre una terminal (CMD, PowerShell o Terminal en Mac) y escribe: node -v y luego npm -v. Deberías ver los números de las versiones instaladas.

Git es el sistema de control de versiones que te permite guardar el historial de tu código. Es la herramienta fundamental que has practicado en los simuladores.

  1. Ve a la página oficial: git-scm.com.
  2. Descarga el instalador para tu sistema operativo.
  3. Ejecuta el instalador. Durante la instalación, te hará muchas preguntas. Para empezar, es seguro aceptar todas las opciones por defecto. La configuración predeterminada es excelente para la mayoría de los usuarios.
  4. Para verificar la instalación, abre una nueva terminal y escribe: code>git --version. Deberías ver la versión de Git instalada.

GitHub es la plataforma donde guardarás tu código en la nube (repositorios remotos). Es el "lugar" al que te conectas cuando usas el comando git push.

  1. Si no tienes una cuenta, crea una gratis en github.com.
  2. Una vez dentro, busca el botón "New" (Nuevo) o un ícono de `+` en la esquina superior derecha y selecciona "New repository".
  3. Repository name: Dale un nombre a tu proyecto (ej. code>mi-primer-proyecto).
  4. Description (Optional): Añade una breve descripción.
  5. Selecciona Public (Público) para que cualquiera pueda verlo (ideal para portafolios).
  6. No marques ninguna de las casillas como "Add a README file", "Add .gitignore" o "Choose a license" por ahora. Es mejor crearlos desde tu computadora.
  7. Haz clic en el botón verde "Create repository".
  8. ¡Listo! GitHub te mostrará una página con la URL de tu repositorio (ej. https://github.com/tu-usuario/mi-primer-proyecto.git). Esta es la URL que usarás en el comando git remote add origin [URL].
